## Core metrics definitions

| Metric | Definition | Notes |
|---|---|---|
| Time to fill | Days from req open to accept | Exclude on-hold periods if tracked |
| Time in stage | Average days per ATS stage | Finds bottlenecks |
| Time to hire | Open to start date | Includes notice period |
| Offer acceptance rate | Accepted / extended | Segment by level |
| Fall-off rate | Declines after verbal/written | Track reason codes |
| Req load | Open reqs per recruiter | Capacity planning |
| Cost per hire | Total recruiting cost / hires | Include agency fees |

Use consistent definitions org-wide; document in TA ops wiki.

## Pass-through and funnel analysis

Calculate pass-through as:

`candidates exiting stage successfully / candidates entering stage`

Review weekly by:

- Req and role family
- Source
- Stage (screen → HM → loop → offer)

**Investigate** when pass-through drops > 10 points vs 90-day baseline:

- JD misaligned to market
- Comp below band
- HM bar drift
- Poor agency quality
- Scheduling delays causing dropout

Pair metrics with **qualitative** HM and candidate feedback.

## Diversity funnel reporting

Report **representation by stage** using org-approved categories and voluntary self-ID:

- Applied → screened → interviewed → offered → hired

Rules:

- Use aggregated data only; protect small-n confidentiality
- Focus on **process fixes** (inclusive JD, diverse slates, trained interviewers)
- Do not set illegal hiring quotas; follow HR/legal guidance on goals vs preferences
- Partner with HR on adverse impact analysis when requested—not TA legal analysis

## Source and channel effectiveness

| Dimension | Questions |
|---|---|
| Volume | Applicants per channel |
| Quality | % reaching HM screen |
| Efficiency | Cost per qualified candidate |
| Hire | % of hires by source |
| Speed | Time-to-fill by source |

Compare **referral**, **direct**, **agency**, **events**, and **sourcer campaigns** (`talent-sourcer`).

Cut spend on channels with high volume and near-zero hire conversion.

## Agency and RPO scorecards

Quarterly scorecard per vendor:

| Category | Weight example |
|---|---|
| Slate quality (screen pass %) | 30% |
| Interview pass % | 25% |
| Time-to-submit vs SLA | 15% |
| Diversity of submissions | 15% |
| Cost per hire | 15% |

Include qualitative: responsiveness, adherence to brief, ethical behavior.

## Executive reporting template

Monthly TA snapshot (one page):

1. **Headline** — hires vs plan, open reqs, critical gaps
2. **Metrics** — time-to-fill trend, offer accept rate, agency spend
3. **Risks** — aging reqs, HM feedback debt, comp misalignment
4. **Actions** — process changes, headcount redeployment, agency changes
5. **Wins** — key hires, improved pass-through, EVP improvements

Avoid data dumps without decisions attached.

## Hiring compliance basics

**Not legal advice**—coordinate with HR/legal for binding guidance.

| Topic | TA habit |
|---|---|
| EEO | Use structured process; coded rejection reasons |
| OFCCP (US federal contractors) | Follow org audit file requirements if applicable |
| Pay transparency | Post ranges where legally required |
| Right to work | Trigger checks only via HR after offer |
| Background checks | FCRA-compliant flow via authorized vendor |
| Accommodations | Route requests to HR; do not deny in recruiting |
| Data privacy | Retain records per retention policy; limit PII sharing |

## Documentation and audit habits

- Store approvals (req, comp, offer) in ATS or linked system
- Keep interview scorecards **before** debrief decisions
- Log **business justification** for rejections using approved reason codes
- Retain agency submissions and feedback for fee disputes
- Run quarterly **process audit** on closed reqs: missing fields, SLA breaches

For employment law questions, route to `hr-business-partner` and counsel—not TA alone.
