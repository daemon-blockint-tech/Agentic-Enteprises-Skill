## Loop design principles

Design loops to measure **job-relevant competencies** with minimum candidate burden:

- **Stage count** — often 3–5 interviews for IC; fewer for high-volume roles
- **Time target** — complete loop within 7–14 calendar days when possible
- **Interviewer training** — bias awareness, legal questions to avoid, scorecard use
- **Panel diversity** — interviewers represent collaboration partners, not clones of HM
- **Coordination** — single recruiter owner for schedule and comms

For question banks and rubric text, load `interview-prep` or `interview-system-designer`.

## Standard stage patterns

| Stage | Purpose | Typical owners |
|---|---|---|
| Recruiter screen | Motivation, logistics, level sanity | Recruiter |
| HM screen | Scope fit, team sell, mutual interest | Hiring manager |
| Technical / functional | Job skills | Peers, senior IC |
| Cross-functional | Collaboration, communication | Partner teams |
| Values / leadership | Culture add, leadership principles | Trained bar raisers |
| Executive (optional) | Senior hires, strategic roles | Director+ |

Document **pass/fail criteria** per stage before interviewing at volume.

## Scorecards and debriefs

1. Assign **competencies** per stage (max 4–6 per interview)
2. Use consistent rating scale (e.g., strong no / no / yes / strong yes)
3. Require **written evidence** before debrief—no memory-only opinions
4. Hold debrief within 24–48 hours of final interview
5. HM decides with recruiter; escalate split panels per policy

Avoid:

- "Culture fit" without defined behaviors
- Debriefs without scorecards completed
- New questions invented mid-loop that other candidates did not receive

## Candidate experience standards

| Touchpoint | Standard |
|---|---|
| Application ack | Auto + human follow-up if qualified within SLA |
| Scheduling | Offer 2–3 time options; timezone clarity |
| Prep materials | Agenda, interviewer names, what to expect |
| Day-of | Start on time; interviewers read resume |
| Post-loop | Update within agreed days—even if "still deciding" |
| Offer | Verbal before written; clear deadline |
| Rejection | Timely, respectful, optional feedback per policy |

Track **candidate NPS** or survey where org uses it.

## Work samples and take-homes

Use paid or **time-bounded** exercises only when predictive:

- Cap time (e.g., 2–4 hours unpaid max; paid beyond org policy)
- Use **realistic** tasks; no unpaid production work
- Same exercise for all candidates at that stage
- Provide rubric to interviewers before review
- Return IP policy clarity if candidate submits code or designs

## Debrief and decision rules

**Advance** when:

- Meets bar on all must-have competencies for stage
- No strong disqualifying evidence
- Level aligned to req

**Hold** when:

- Missing one competency—schedule targeted follow-up once
- Panel split—third interviewer or HM deep-dive

**Reject** when:

- Fails must-have with evidence
- Level mismatch not fixable by req change
- Values or integrity concern with documented facts

Record decision and **business reason** in ATS per compliance habits.

## Rejection and withdrawal

**Rejection:**

- Send within org SLA (e.g., 3–5 business days after decision)
- Thank candidate; avoid false hope language
- Offer feedback only if policy allows and interviewer trained

**Withdrawal:**

- Document reason (comp, role, competing offer, ghost)
- Analyze patterns (slow process, low comp, bad JD)
- Re-engage silver medalists when req reopens if policy allows
