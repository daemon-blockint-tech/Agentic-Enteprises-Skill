# Portfolio Charter

## Charter elements

| Element | Content |
|---|---|
| **Mission** | Why the DC portfolio exists (capacity, latency, sovereignty, cost) |
| **Scope** | Sites, regions, in/out (edge, DR only, etc.) |
| **Horizon** | Planning window (e.g. FY26–FY30) |
| **Budget envelope** | Capex bands, opex run-rate targets |
| **Success metrics** | e.g. zero unplanned capacity breaches, PUE trend, delivery on milestone % |
| **Governance** | Steering cadence, voting members, escalation |

## Roles

| Role | Responsibility |
|---|---|
| **Portfolio lead** | Roadmap, prioritization, steering pack |
| **Site / program DRI** | Single hall or expansion delivery |
| **Facilities** | Power, cooling, physical install |
| **IT infrastructure** | Compute, network, handoff to ops |
| **Finance** | Capex approval, TCO model |
| **Sustainability** | Carbon and water targets |
| **Security / compliance** | Physical and audit scope |

Delegate site design depth to `data-center-design-execution-lead` DRIs.

## Decision rights

| Decision type | Typical forum |
|---|---|
| New region or tier change | Steering + exec |
| Prioritize initiative A over B | Steering |
| Single-site MEP detail | Site DRI + facilities |
| Defer refresh one quarter | Portfolio lead + finance |

## Out of scope (document explicitly)

- Application feature roadmaps
- Cloud account structure (unless tied to DC exit)
- Generic engineering hiring

## Handoff from strategy

If `business-consultant` produced options, translate into **funded initiatives** with owners and dates—not restate strategy slides.
