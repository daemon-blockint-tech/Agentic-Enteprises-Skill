# Closing Matrix

## Matrix columns

| Column | Description |
|---|---|
| **Item** | Document, filing, or action |
| **Category** | Signature / CP / Deliverable / Filing / Payment |
| **Party** | Buyer, seller, counsel, secretary |
| **Responsible** | Name + org |
| **Status** | Not started / Draft / Agreed / Executed / Filed / N/A |
| **Target date** | |
| **Dependencies** | Other row IDs |
| **Notes** | Version, escrow link |

## Conditions precedent (CPs)

CPs must be **objectively verifiable**:

| CP type | Example evidence |
|---|---|
| Regulatory | Approval letter, expiration of waiting period |
| Third party | Signed consent |
| Corporate | Board/stockholder resolutions (`corporate-counsel`) |
| Financial | Payoff letter, release |
| Litigation | Dismissal order |

**CP waiver** — document committee approval and counterpart acknowledgment.

## Bring-down and certificates

- Officer certificates dated as of closing
- Secretary certificates and incumbency
- Good standing (dated within agreed window)
- Legal opinions — dependencies listed explicitly

Coordinate substance with `corporate-counsel`; transaction manager tracks **status**.

## Disclosure schedules

- Schedule updates through signing
- Cross-reference diligence findings
- Final **schedule bring-down** at closing

## Signing vs closing

| Event | Typical contents |
|---|---|
| **Signing** | Definitive agreements, some ancillaries |
| **Closing** | Funds, filings, remaining CPs |

Some deals sign and close same day; others gap days/weeks — matrix must reflect **both dates**.

## RAG per row

| RAG | Rule |
|---|---|
| Green | On track for target date |
| Amber | At risk; mitigation owner assigned |
| Red | Blocking close; escalated |
