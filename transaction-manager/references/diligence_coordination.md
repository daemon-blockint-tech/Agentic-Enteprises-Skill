# Diligence Coordination

## Workstreams

| Workstream | Typical scope | Lead |
|---|---|---|
| **Legal** | Corporate, contracts, litigation, IP | External + `corporate-counsel` / `commercial-counsel` |
| **Financial** | QoE, working capital, debt | External accountants |
| **Tax** | Structure, exposures, elections | Tax advisers |
| **HR** | Benefits, comp, key talent | HR + `people-operations-specialist` |
| **IT / security** | Systems, cyber, data | IT + `compliance-engineer` |
| **Commercial** | Customers, pipeline, churn | Business + sales |
| **Regulatory** | Licenses, compliance | Compliance |

Transaction manager **coordinates**; workstream leads **own content**.

Rules:

- One **ID** per request; no duplicate threads
- **Aging** >5 business days without response → escalate in steering
- Material findings → link to disclosure schedule or price adjustment track

## Data room

| Practice | Why |
|---|---|
| Folder taxonomy by workstream | Faster reviewer navigation |
| Index with doc IDs | Closing binder continuity |
| Version naming (v1, v2) | Avoid wrong doc signed |
| Access log | Confidentiality |
| Redaction policy | PII, third-party secrets |

Close loop before definitive agreement sign-off where possible.

## Diligence summary (weekly)

1. **Progress** — % requests complete by workstream
2. **Top findings** — severity, mitigation, price/structure impact
3. **Blockers** — access, timing, adviser capacity
4. **Decisions** — proceed, retrade, walk

Do not substitute for legal or accounting opinions in summary.
