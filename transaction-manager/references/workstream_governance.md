# Workstream Governance

## Deal RAID (portfolio)

| Type | Example | Owner |
|---|---|---|
| **Risk** | HSR second request extends timeline | Regulatory |
| **Issue** | Data room missing customer contracts | Commercial |
| **Action** | Obtain payoff letter by 3/15 | Finance |
| **Decision** | Accept locked-box vs completion accounts | Deal committee |

Site-level legal analysis stays with counsel — RAID tracks **impact on timeline and economics**.

## Steering pack (weekly)

1. **Milestone** — vs plan (sign, close)
2. **Matrix summary** — % complete, red rows
3. **Diligence** — open high-severity findings
4. **Decisions** — numbered with recommendation
5. **Budget** — adviser fees vs cap

## Decision log

| Date | Decision | Approver | Impact |
|---|---|---|---|

Required for: price adjustment, structure change, waiver, extension, walk-away.

Transaction manager tracks **deliverables and invoices**; counsel selects legal strategy.

## Confidentiality

- Clean team / segregated personnel if required
- Code names in calendars and file shares
- No material terms in unsecured chat

## Escalation

| Trigger | Escalate to |
|---|---|
| Outside date within 14 days and CPs open | Deal committee |
| Material adverse finding | Deal committee + counsel |
| Workstream lead unavailable | Exec sponsor |

## Differentiation from TPM

| Transaction manager | Technical program manager |
|---|---|
| Deal-specific CPs, signing, funds | Product/engineering milestones |
| Diligence and disclosure | Dependencies between engineering teams |
| Ends at integration handoff | May run quarters post-close |

Use both when integration is large: transaction manager through close, TPM for integration program.
