# Program Charter and Governance

## Charter elements

| Section | Content |
|---------|---------|
| Mission | Why exec + community escalation program exists |
| Scope | In / out (e.g. in: VIP, strategic, public viral; out: routine L1 tickets) |
| Objectives | Measurable (SLA adherence, exec satisfaction, repeat rate) |
| Stakeholders | CS, support, product, eng, legal, comms, exec sponsors |
| Forums | Daily war room, weekly steering, monthly metrics review |
| RACI | Per activity (see below) |
| Escalation ladder | When to involve VP, C-suite, CEO |

## RACI (typical)

| Activity | Program PM | CS | Support | Eng | Product | Legal | Comms | Exec sponsor |
|----------|------------|----|---------|-----|---------|-------|-------|--------------|
| Intake & triage | A/R | C | R | I | I | I | I | I |
| War room chair | R/A | C | C | C | C | C | C | I |
| Exec brief | R | C | C | C | C | C | C | A |
| Customer commit | C | A/R | C | C | C | C | C | I |
| Public statement | C | I | I | I | I | A | R | A |
| Playbook update | R/A | C | C | C | C | C | C | I |

R = responsible, A = accountable, C = consulted, I = informed.

## Escalation ladder (example)

| Level | Trigger | Forum | Decision authority |
|-------|---------|-------|-------------------|
| L1 | Standard strategic account issue | Account team + support lead | CS director |
| L2 | SLA breach, multi-team blocker | War room | Program sponsor |
| L3 | Revenue-at-risk, legal exposure, viral community | Exec briefing | VP or C-suite DRI |
| L4 | Board-visible, regulatory, widespread outage + reputational | CEO staff | CEO + GC + CCO |

Tune thresholds to company stage.

## Policies to document

- **Single front door** for exec escalations (email alias, form, CRM flag)
- **No surprise execs** — brief sponsor before external exec outreach
- **Document retention** — where notes, decisions, and comms live
- **Confidentiality** — what can be shared in community vs 1:1
- **Sunset** — when case exits program (resolved, downgraded, transferred)

## Steering committee cadence

| Meeting | Audience | Inputs | Outputs |
|---------|----------|--------|---------|
| Weekly | Directors + PM | Open queue, reds, SLA | Priorities, resource asks |
| Monthly | VP + PM | Trends, playbook gaps | Policy changes, headcount |

## Anti-patterns

- Program PM owns technical root cause (delegate to eng DRI)
- Parallel exec paths bypassing triage
- Permanent war rooms with no exit criteria
